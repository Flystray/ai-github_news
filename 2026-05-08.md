# GitNexus：让 AI 真正"看懂"你的代码库

> GitHub 3.7 万 Stars | TypeScript | 2025-08 发布 | 完全浏览器端运行
> https://github.com/abhigyanpatwari/GitNexus

## 项目背景

AI 编程助手（Cursor、Claude Code、GitHub Copilot 等）已经能帮我们写代码、改 Bug，但它们有一个共同的盲区：**不理解代码库的整体结构**。

当你让 AI "修改用户认证逻辑"时，它往往不知道这个改动会影响哪些模块、哪些调用链会中断、哪些测试需要同步更新。结果就是：AI 改了一处，别处悄悄崩了。

这正是 GitNexus 要解决的核心问题。

2025 年 8 月，开发者 abhigyanpatwari 开源了 GitNexus——一个**零服务器的代码智能引擎**。它的独特之处在于：把整个代码库构建成知识图谱，让 AI Agent 在回答之前就能"看见"完整的依赖关系网络。

更令人惊讶的是，这一切完全在浏览器或本地机器上运行，代码永远不会离开你的设备。

## 核心功能

### 1. 代码知识图谱自动构建

GitNexus 最强大的地方在于**预计算智能**——它不是让 LLM 实时去探索代码关系，而是在索引阶段就把所有关系算好了：

- 遍历文件树，映射文件夹/文件层级关系
- 用 Tree-sitter AST 解析函数、类、方法、接口
- 跨文件解析导入依赖、函数调用链、继承关系
- 将相关符号聚类为功能模块
- 从入口点追踪完整执行流程

最终生成的知识图谱包含：每个符号（函数/类/变量）的位置、类型、调用者、被调用者、所属模块、置信度评分。

### 2. 两种使用模式

#### **CLI + MCP 模式**（推荐日常开发）

```bash
# 在项目根目录运行，一键索引
npx gitnexus analyze
```

这条命令会自动完成：
- 索引整个代码库（支持任意规模）
- 安装 Agent Skills 到 `.claude/skills/`
- 注册编辑器 MCP 连接
- 生成 `AGENTS.md` / `CLAUDE.md` 项目上下文文件

配合 Claude Code、Cursor、Windsurf 等编辑器使用时，AI 会自动调用 GitNexus 的 MCP 工具来获取代码上下文，**一次调用就能拿到完整依赖视图**，不需要让 AI 自己一步步探索。

#### **Web UI 模式**（适合快速探索）

直接访问 [gitnexus.vercel.app](https://gitnexus.vercel.app)，拖入 GitHub 仓库 URL 或上传 ZIP 文件，即可在浏览器内查看交互式知识图谱 + 与 Graph RAG Agent 对话。

- 无需安装任何软件
- 代码完全在浏览器内处理，不会上传到服务器
- 受浏览器内存限制（约 5000 个文件），或通过后端模式解除限制

### 3. 为 AI Agent 量身打造的 16 个 MCP 工具

GitNexus 通过 MCP（Model Context Protocol）向 AI 编辑器暴露了 16 个强大的代码理解工具：

| 工具 | 功能 |
|------|------|
| `query` | 混合搜索（BM25 + 语义 + RRF 融合），快速定位相关代码 |
| `context` | 360° 符号视图，查看一个函数的所有调用者/被调用者 |
| `impact` | **爆炸半径分析**，改动一个函数会影响哪些地方，附置信度 |
| `detect_changes` | Git diff 影响分析，PR 提交前预判风险 |
| `rename` | 多文件协调重命名，自动追踪所有引用点 |
| `cypher` | 原始 Cypher 图查询，高级用户自定义查询 |

以及 5 个组级工具，用于管理多个仓库的统一图谱。

### 4. 4 个自动安装的 Agent Skills

运行 `npx gitnexus analyze` 后，你的项目里会自动装上 4 个 Skills：

- **探索（Exploring）** — 用知识图谱导航陌生代码库，快速上手新项目
- **调试（Debugging）** — 通过调用链追踪 Bug 根因，不再盲目搜索
- **影响分析（Impact Analysis）** — 改动前先算爆炸半径，防止引入破坏性变更
- **重构（Refactoring）** — 基于依赖映射规划安全重构路径

## 技术亮点

### 预计算关系智能：为什么比传统 Graph RAG 更高效？

传统 Graph RAG 的做法是：LLM 收到问题 → 查询图数据库 → 遍历边 → 收集上下文 → 生成答案。这个过程中 LLM 需要多次往返查询，Token 消耗大，且容易遗漏关键路径。

GitNexus 的做法是**在索引阶段就把关系算好**：

- 聚类（哪些函数属于同一个功能模块）
- 调用链追踪（从入口到终端的完整执行路径）
- 置信度评分（这个调用关系的确定性有多高）

当 AI 调用 `query` 工具时，GitNexus 直接返回**预处理好的完整上下文**，不需要 LLM 自己探索图。

**实际效果**：
- 较小的 LLM（如 GPT-4o-mini）也能完成复杂的代码理解任务
- Token 消耗大幅降低（不需要多轮探索）
- 不会遗漏关键上下文（已经在工具响应里了）

### 多仓库统一图谱

GitNexus 使用全局注册表（`~/.gitnexus/registry.json`）管理所有已索引的仓库。一个 MCP 服务器可以同时服务多个仓库，AI Agent 可以跨仓库理解代码关系。

```bash
# 列出所有已索引的仓库
npx gitnexus list

# 启动本地 HTTP 服务器，让 Web UI 连接所有已索引的仓库
npx gitnexus serve
```

### Docker 一键部署

```bash
# 使用官方 Docker 镜像部署服务端
docker compose up -d
# Web UI: http://localhost:4173
# API 服务: http://localhost:4747
```

官方镜像使用 Cosign 无密钥签名，附带 SBOM 证明，可用于 Kubernetes 准入策略强制验证。

## 实际应用场景

### 场景 1：上手陌生代码库

> "这个开源项目是怎么组织的？主流程从哪里开始？"

用 GitNexus Web UI 打开仓库，知识图谱会展示完整的模块划分和调用关系。比直接读 README 或盲目 grep 高效得多。

### 场景 2：修改前的影响分析

> "如果我改了这个函数，会有什么影响？"

调用 `impact` 工具，GitNexus 会返回完整的爆炸半径分析报告：直接影响（调用方）、间接影响（调用方的调用方）、置信度评分。

### 场景 3：PR 代码审查

GitNexus 可以自动对 Pull Request 进行影响分析，生成审查报告：这次改动影响了哪些模块、是否有测试覆盖、是否存在破坏性变更风险。

### 场景 4：多文件重构

> "把 `UserService` 重命名为 `AccountService`"

用 `rename` 工具，GitNexus 会自动找到所有引用点（包括跨文件的字符串引用），一次性安全重命名。

## 快速上手

### 方式一：CLI + MCP（推荐）

```bash
# 1. 索引当前项目
npx gitnexus analyze

# 2. 配置编辑器 MCP（仅首次需要）
npx gitnexus setup

# 3. 启动 MCP 服务器
npx gitnexus mcp
```

### 方式二：Web UI（零安装）

1. 打开 [gitnexus.vercel.app](https://gitnexus.vercel.app)
2. 输入 GitHub 仓库 URL 或上传 ZIP
3. 等待索引完成（通常在 10~60 秒内）
4. 开始与 Graph RAG Agent 对话探索代码

### 方式三：Docker 部署（团队协作）

```bash
git clone https://github.com/abhigyanpatwari/GitNexus.git
cd GitNexus
docker compose up -d
```

## 总结

GitNexus 解决了一个被大多数人忽视但极其重要的问题：**AI 编程助手在修改代码时，往往不知道自己改坏了什么**。

通过构建代码知识图谱并预计算关系，GitNexus 让 AI Agent 真正"看见"代码库的全貌。它的零服务器架构（完全本地/浏览器内运行）也意味着你可以放心地用它分析私有代码库，不用担心代码泄露。

如果你正在使用 Claude Code、Cursor 等 AI 编程工具，GitNexus 会是一个值得尝试的强力补充。
