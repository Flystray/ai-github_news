### 📌 项目背景

你有没有过这种经历——明明昨天看过一个重要信息，翻遍浏览器历史也找不到，只能干着急？或者开会时来不及记笔记，回头想整理行动项却一片空白？

这类问题催生了"桌面 AI 记忆"这个赛道。Rewind.ai 是最早的探索者，但它后来转向了云端付费产品并最终停止维护。screenpipe 接过了这面旗帜，用完全开源、本地优先的方式重新定义了"个人 AI 记忆"——它的核心理念是：**你的电脑应该比你更清楚你做过什么**。

screenpipe 由 Rust 语言构建（强调性能与安全），于 2024 年 6 月发布，至今已获得 18.5k GitHub stars 和 1.7k forks，项目保持高度活跃（每天均有 push），是当前同类开源项目中社区最活跃的一个。与传统录屏工具不同，screenpipe 的价值不在于"录像"，而在于让 AI 真正理解屏幕上的内容并采取行动。

---

### 🚀 核心功能

**1. 24/7 全天候屏幕 + 麦克风录制**

screenpipe 在后台持续记录屏幕画面和麦克风音频。与普通录屏软件不同，它采用"事件驱动录制"策略——只在检测到有意义的变化（切换应用、鼠标点击、键盘输入、滚动）时才保存截图，而非每秒机械录制。这使得存储效率大幅提升，同时不会遗漏关键信息。

**2. 多模态 AI 搜索：问 AI"我昨天看了什么"**

录制的内容会自动经过 OCR 文字识别、音频转录等多模态处理，存入本地 SQLite 数据库。用户可以用自然语言直接查询，例如：
- "过去 5 分钟我在看什么？"
- "这周我和谁讨论过定价问题？"
- "总结今天所有会议的要点和行动项"

这种搜索能力远超传统关键词搜索，因为 AI 理解的是屏幕内容的语义。

**3. Pipes 自动化系统：基于桌面活动的 AI Agent 工作流**

这是 screenpipe 最具创新性的功能。**Pipes** 是以 Markdown 文件定义的定时 AI Agent——你可以把一个 pipe.md 配置为"每天下午 5 点自动查询今天的屏幕记录，总结工作进度并更新到 Linear"。

pipe 的触发条件非常灵活：
- 定时触发（如每天早上 9 点）
- 事件触发（如会议结束时）
- 应用触发（如打开某个 App 时）

目前社区已共享了大量预构建 pipes，包括：自动生成会议笔记并保存为 Markdown、追踪每日 ideas、自动同步到 Notion/Obsidian、定时提醒等。

**4. MCP 协议集成：让 AI 编码助手读懂你的屏幕**

screenpipe 将搜索 API 暴露为 MCP 服务器（Model Context Protocol），使得 Claude Desktop、Cursor、Cline、Continue 等主流 AI 编码助手可以直接查询用户的屏幕和音频历史。

安装方式极为简单：
```bash
npx screenpipe@latest record
claude mcp add screenpipe -- npx -y screenpipe-mcp
```

集成后，你可以直接问 Claude："我昨天写的 API 接口在哪里？""过去一周我和谁讨论过这个 bug？"，AI 会结合屏幕历史给出精准答案。这对于代码审查、知识管理场景价值巨大。

**5. 48 个第三方应用集成（Connections）**

screenpipe 支持与 48 个主流应用深度集成，覆盖：
- **通讯**：Slack、Discord、Telegram、WhatsApp、Zoom、Google Meet
- **邮件/日历**：Gmail、Google Calendar
- **笔记/文档**：Notion、Google Docs/Obsidian、Confluence
- **项目管理**：Linear、Jira、Asana、Monday、Trello
- **CRM**：HubSpot、Salesforce、Pipedrive
- **开发工具**：GitHub、GitLab、Sentry、Datadog
- **设计**：Figma、Miro、Loom
- **AI**：OpenAI、Claude、Ollama、Gemini

连接一次后，所有 Pipes 自动化工作流均可自动调用这些集成。

**6. 三种录制策略：会议模式 / 编程模式 / 休息模式**

用户可根据场景灵活切换录制策略：
- **会议模式**：高帧率屏幕 + 全质量音频，完整记录每一个讨论细节
- **编程模式**：重点捕获代码编辑区域，降低视频质量以节省存储
- **休息模式**：自动暂停录制，保护个人隐私

### ⚙️ 技术亮点

**1. Rust 语言全栈本地架构，性能与隐私兼得**

screenpipe 全链路采用 Rust 构建（核心层 + Tauri 桌面封装），CPU 占用仅 5-10%，内存占用 0.5-3GB，每月存储约 20GB。数据从捕获到处理到存储全部在本地完成，没有任何数据上传到远程服务器。这是真正的"隐私优先"架构，而非营销话术。

**2. 三层 AI 集成架构：MCP + Chat UI + Pipes**

screenpipe 提供了三种不同层次的 AI 访问方式：
- **MCP 服务器层**：供 Claude Desktop、Cursor 等外部 AI 助手调用（标准 JSON-RPC over stdio）
- **Chat UI 层**：screenpipe 自带对话界面，支持多 AI 模型切换（Claude Opus 4.7、Gemma 4 Confidential、Qwen 3.5 等）
- **Pipes Agent 层**：后台自动化执行，通过 `screenpipe-search` skill 查询数据并执行动作

三层架构满足了从实时对话到后台自动化的全场景需求。

**3. 中文 OCR + 本地 LLM 支持（Ollama 集成）**

screenpipe 内置强大的 OCR 引擎，支持中文在内的多语言文字识别。更重要的是，它可以完全接入 Ollama，在本地运行 Llama、Qwen 等开源大模型，实现**完全离线**的 AI 助手体验——不需要任何云端 API，数据永远不会离开你的设备。

**4. 可插拔的 Filter 系统：隐私安全保护**

内置多层隐私过滤器：
- 按窗口、按应用、按 Chrome 扩展过滤录制内容
- 自动过滤密码输入框、PII（个人身份信息）内容
- 可选端到端加密存储

### 👥 适用人群

**独立开发者 / 程序员**：screenpipe 的 MCP 集成让你在 Cursor 或 Claude Desktop 中可以直接搜索自己写过哪些代码、和同事讨论过哪些技术方案，大幅提升代码复用率。对于长期项目，screenpipe 充当了一个永不遗忘的知识库。

**远程团队成员 / 频繁开会者**：会议结束后自动生成结构化会议纪要（3 条要点 + 每人行动项 + 决议），无需手动整理。可以自动同步到 Linear、Jira 等项目管理工具。

**知识工作者 / 研究者**：当你需要回顾"两周前我在哪个网站看到过这篇论文"或"我和客户讨论定价策略的那封邮件"时，screenpipe 的自然语言搜索比任何浏览器历史都高效。

**注重隐私的用户**：所有数据本地存储，支持 Ollama 完全离线运行，不依赖任何云服务，是 Rewind.ai、微软 Recall 的真正开源替代。

### ⚡ 快速上手

**桌面 App（一键安装）：**

访问 [screenpi.pe/onboarding](https://screenpi.pe/onboarding) 下载桌面应用，授权屏幕和麦克风权限，选择存储位置，即可开始录制。

**CLI 方式（开发者首选）：**
```bash
# 安装
npx screenpipe@latest record

# 集成 Claude Desktop
claude mcp add screenpipe -- npx -y screenpipe-mcp

# 开始使用
# 问 Claude："what did i see in the last 5 mins?"
# 或："summarize today's conversations"
```

**安装后首次配置建议：**
1. 授予屏幕和麦克风权限
2. 选择数据存储路径（默认 `~/.screenpipe/data`）
3. 配置 Ollama（可选，用于完全离线模式）
4. 安装感兴趣的应用集成（Notion、Linear 等）
